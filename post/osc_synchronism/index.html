多个进程并发的操作同一个变量并且结果与执行顺序有关, 被称为竞争条件. 会导致结果不确定. 为了有正确的结果, 需要进程造成的改变不相互干扰

临界区问题

假设某个系统有 n 个进程, 它们有一段代码操作公共数据, 这一段代码被称为临界区(critical section), 为了进程协调, 不能有两个进程同时执行临界区代码. 为了实现这一目标, 进程进入临界区时应申请许可, 申请许可的代码被称为进入区(entry section), 临界区之后还有退出区(exit section). 其它代码为剩余区(remainder section). 解决方案应满足

  1. 互斥(mutual exclusion)
  2. 前进(progress): 当没有进程正在执行临界区代码, 且有进程需要进入临界区, 那些不在剩余区执行的进程可以被选择
  3. 有限等待(bounded waiting)

Peterson

由于现代计算机执行基本机器指令的方式不同, Peterson 方案以经不能确保正确, 不过该方案的思想与算法还是不错的. 该方案适用于两个进程交错执行临界区和剩余区代码的情况, 两个进程为 \(P_0\) 和 \(P_{1}\) . Peterson 要求两个进程共享两个数据项:

信号量

一个信号量(semaphore) S 是个整型变量, 除了被始化外只能通过两个原子操作: wait() , signal() 来访问. 操作系统通常区分计数信号量(counting semaphore)和二进制信号量(binary semaphore). 二进制信号量只能是 0 或 1, 类似于互斥锁.

信号量使用

用信号量表示资源数目, 当进程需要资源时执行 wait() 操作, 如果资源大于 0, 就将资源减 1, 否则阻塞. 使用完资源调用 signal() 将信号量加 1.

信号量的实现

之前的信号量有忙等待, 可以进行改进. 调用 wait() 时发现信号量非正, 进程不是忙等待, 而是将自己放入与信号量相关的队列中并将自己状态改为等待. 当有进程调用 signal() 释放资源时, 需要调用 weakup() 重新执行等待的进程. 可以这样定义信号量

typedef struct {
    int value;
    struct process* list;
} semaphore;

/*  对应的 wait() 和 signal()  */

wait(semaphore S)
{
    S->value--;
    if (S->value < 0)
    {
        add this process to S->list;
        block();
    }
}

signal(semaphore S)
{
    S->value++;
    if (S->value <= 0)
    {
        remove a process P from S->list;
        weakup(P);
    }
}

死锁和饥饿

两个或多个进程等待一个事件, 而该事件必须由这些被阻塞的进程产生, 就会产生死锁.

饥饿是指无限等待, 如采用后进先出的顺序时, 先进栈的进程可能会被一直阻塞

优先级反转

如果一个进资源被一个低优先级的进程使用, 一个高优先级的进程就需要等待低优先级进程用完资源. 如果低优级的进程被其它中等优先进程抢占, 高优先级的进程就会被中等优先的进程卡住. 这时需要一个优先级继承协议, 进程使用资源时继承想使用该资源进程中最高的优先级.

monitor DiningPhilosophers
{
    enum { THINKING, HUNGRY, EATING } state[5];
    condition self[5];
    void pickup(int i)
    {
        state[i] = HUNGRY;
        test(i);
        if (state[i] != EATING) self[i].wait();
    }
    void putdown(int i)
    {
        state[i] = THINKING;
        test((i + 4) % 5);
        test((i + 1) % 5);
    }
    void test(int i)
    {
        if (state[(i + 4) % 5] != EATING && state[(i + 1) % 5] != EATING && state[i] == HUNGRY)
        {
            state[i] == EATING;
            self[i].signal();
        }
    }

    initialization_code()
    {
        for (int i = 0; i < 5; ++i) state[i] = THINKING;
    }
}

/* 共享数据结构 */
boolean flag[2];
int turn;
do
{
    flag[i] = true;
    while (flag[j])
    {
        if (turn == j)
        {
            flag[i] = false;
            while (turn == j)
                ;
            flag[j] = true;
        }
    }
    // ...
    turn    = j;
    flag[i] = false;
} while (1);
  • 互斥: 若 P_{j} 正在临界区执行则 flag[j]==true, P_{i} 会不断执行 while(flag[j]) 语句, 不会进入临界区, 反之亦然. 若 P_{i}, P_{j}同时到达进入区, turn 只可能是 i 和 j 其中之一, 也就只有一个进程可以完成 while(turn == x) 语句. 互斥成立
  • 进步: 若 P_{j} 退出临界区, 会设置 turn = i; flag[j] = false. 若 P_{i} 想要进入临界区, 其可以进入
  • 有限等待: P_{i} 等待 P_{j} 执行一次临界区后即可进入

int waiting[n];

do
{
    waiting[i] = 1;
    key        = 1;
    while (waiting[i] && key) key = compare_and_swap(&lock, 0, 1);
    waiting[i] = false;

    /*  critical section */

    j = (i + 1) % n;
    while (j != i && !waiting[j]) j = (j + 1) % n;
    if (j == i)
        lock = 0;
    else
        waiting[j] = 0;
  /* remainder section */
} while (1);

#define MAX_RESOURCES 5
int available_resources = MAX_RESOURCES;
semaphore mutex;
int decrease_count(int count)
{
    if (available_resources < count)
        return -1;
    else
    {
        wait(mutex);
        available_resources -= count;
        signal(mutex);
        return 0;
    }
}
int increase_count(int count)
{
    wait(mutex);
    available_resources += count;
    signal(mutex);
    return 0;
}
