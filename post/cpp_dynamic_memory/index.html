动态内存和智能指针

动态内存使用一对运算符: new, delete. 动态内存很容易出问题, 比如说没有在正确时间释放内存导致内存泄漏, 或者使用了指向已经释放了的内存, 为了使动态内存更简单和安全. 库提供了两种智能指针

  • shared_ptr: 允许多个指针指向同一个对象
  • unique_ptr: 独占指向的对象

一个配合的类 weak_ptr: 是一个被 shared_ptr 管理对象的弱引用

shared_ptr

shared_ptr 是一个模版类.

1
2
3
4
5

shared_ptr<string> pStr;
shared_ptr<list<int>> pLst;

if(pStr && pStr->empty())
    *pStr = "Hei";

make_shared 函数

1
2
3
4

shared_ptr<string> p1      = make_shared<string>("999999");
shared_ptr<string> p2      = make_shared<string>(10, '9');
shared_ptr<vector<int>> p3 = make_shared<vector<int>>(10, 9);
shared_ptr<int> p3         = make_shared<int>(42);

shared_ptr 拷贝和赋值

拷贝和赋值 shared_ptr 时, 指向同一对象的所有 shared_ptr 会追踪有多少指针指向该对象. 可以认为其有一个相关的计数器. 当其被拷贝或用来初使化另一个 shared_ptr 时计数器会加 1, 当 shared_ptr 被赋新值时或被销毁时, 原计数器减 1. 计数器为 1 时自动释放管理对象的内存

shared_ptr 自动销毁理管理对象

生存周期到了后调用析构函数

直接管理内存

  • new: 分配内存
  • delete: 释放 new 分配的内存

内存枯竭

虽然现代机器有非常在的内存, 自由存储区仍然有可能耗尽. 如果一个程序用尽了允许的内存, new 会失败, 默认抛出 bad_alloc 异常, 可以使用另一种 new 禁止抛出异常

1
2

int* p1 = new int;            // if failed, throw bad_alloc
int* p2 = new (nothrow) int;  // if failed, return nullptr;

释放内存

delete 指针, 销毁对象, 释放内存释放栈对象或已释放内存是未定义行为. delete 空指针是合法的

delete 后重设指针

当 delete 一个指针时后, 指针仍指向已释放的内存, 称为悬挂指针(dangling pointer). 为了避免可以在相关指针本身将过期时删除, 或者将 nullptr 赋值给该指针. 可惜这种保护很有限. 如果有多个指针指向该元素, 其它指针仍然不安全

shared_ptr with new

T* q = new T;

  • shared_ptr p(q)
  • shared_ptr p(u): u 是 unique_ptr, 之后 u 是 null, 其管理的对象交给 shared_ptr

不要混用内建指针和智能指针

容易出错

不要使用 get 获得的内建指针初始化另一个智能指针

容易出现悬挂指针

智能指针和异常

智能指针确保不用的内存会释放, 即使提前结束代码块. 而内建指针会在抛出异常且没有在 delete 之前捕获, 会造成内破存泄漏

智能指针与 dumb classes

许多类定义了析构函数负则释放对象资源, 不过通常来说, 同时为 c 和 c++ 设计的类需要专门释放资源, 这种类有可能出现抛出异常后导致资源泄漏. 可以使用智能指针

unique_ptr

unique_ptr 独占指向的对象, 与 shared_ptr 不同, 一个对象同时只能被一个 unique_ptr 指向. 不能拷贝和赋值, 通过绑定 new 返回的指针初始化. 虽然不能拷贝和赋值, 可以通过 reset, release 来转移所有权

1
2
3
4

unique_ptr<string> p1(new string("hi"));
unique_ptr<string> p2(p1.release());  // p1 will be null;
unique_ptr<string> p3(new string("hello"));
p2.reset(p3.release());

传递和返回 unique_ptr

不能拷贝 unique_ptr 有一个例外, 就是被拷贝对象将被销毁

1

auto_ptr 类似于 unique_ptr 但是不安全且不如用, 建议新程序用 unique_ptr 替换

weak_ptr

weak_ptr 不控制所指对象的生命期. 其所指对象由 shared_ptr 管理, weak_prt 不会改变 shared_ptr 引用计数. 如果 shared_ptr 引用计数为 0, 管理对象会被释放, 即使有 weak_ptr 正指向该对象

操作 描述
weak_ptr<T> w null 指针, 能指向类型 T
weak_ptr<T> w(sp) 指向 shared_ptr sp 管理对象
w = p p 可以是 weak_ptr 也可以是 shared_ptr
w.reset() 置空
w.use_count() 引用计数
w.expired() 引用计数为 0 时返回 true
w.lock() return shared_ptr

#ifndef _BLOB_HPP_
#define _BLOB_HPP_
#include <initializer_list>
#include <memory>
#include <stdexcept>
#include <string>
#include <vector>

template <typename T>
class BlobPtr;
template <typename T>
class Blob {
public:
    friend class BlobPtr<T>;
    BlobPtr<T> begin()
    {
        return BlobPtr<T>(*this);
    }
    BlobPtr<T> end()
    {
        return BlobPtr<T>(*this, data->size());
    }
    using size_type = size_t;
    Blob<T>() : data(std::make_shared<std::vector<T>>()) {}
    Blob<T>(std::initializer_list<T> il) : data(std::make_shared<std::vector<T>>(il)) {}
    size_type size() const
    {
        return data->size();
    }
    bool empty() const
    {
        return data->empty();
    }
    void push_back(const T& _v)
    {
        data->push_back(_v);
    }

    void pop_back()
    {
        check(0, "pop_back on empty Blob");
        data->pop_back();
    }
    T& front()
    {
        check(0, "front on empty Blob");
        return data->front();
    }

    const T& front() const
    {
        check(0, "front on empty Blob");
        return data->front();
    }

    T& back()
    {
        check(0, "back on empty Blob");
        return data->back();
    }

    const T& back() const
    {
        check(0, "back on empty Blob");
        return data->back();
    }

private:
    std::shared_ptr<std::vector<T>> data;
    void check(size_type i, const std::string& msg) const
    {
        if (i >= data->size()) throw std::out_of_range(msg);
    }
};
template <typename T>
class BlobPtr {
public:
    using size_type = size_t;
    BlobPtr<T>() : curr(0){};
    BlobPtr<T>(Blob<T>& b, size_type sz = 0) : data(b.data), curr(sz){};
    T& operator*() const
    {
        auto p = check(size, "dereference past end");
        return (*p)[curr];
    }
    BlobPtr<T>& operator++()
    {
        check(curr, "increment past end of BlobPtr");
        ++curr;
        return *this;
    }
    BlobPtr<T> operator++(int)
    {
        check(curr, "increment past end of BlobPtr");
        auto temp = *this;
        ++curr;
        return temp;
    }

private:
    std::shared_ptr<std::vector<T>> check(size_type i, const std::string& msg) const
    {
        auto ret = data.lock();
        if (!ret) throw std::runtime_error("己释放");
        if (i >= ret->size()) throw std::out_of_range(msg);
        return ret;
    }
    std::weak_ptr<std::vector<T>> data;
    size_type curr;
};
#endif

指针操作

动态数组

new 和 delete 一次操作一个对象. 有时需要一次为多个对象分配内存. 为了这种情况, 有两种方法一次申请一个对象数组. 定义了第二种 new 表达式. STL 中有模版类 allocator.

很多甚至大部分情况不需要直接使用动态数组, 可以使用标准库容器, 如 vector, 来替代. 新的标准库也非常的快速, 安全, 简单. 容器可以使用默认拷贝, 赋值, 析构函数. 如果使用动态数组, 这三种函数需要自己定义.

new 和数组

使用 new 分配一组对象, 需要在类型后用方括号输入大小

1

int * pia = new int[128];

方括号中的数据必须是整数类型(char, short, int, long, long long…), 不过不需要是常量

new 数组产生一个指向元素的指针

new 分配数组时, 我们无法获取数组类型. 我们获取的是指针. 不能使用 begin, end. 不能使用 range for. 使用 sizeof 得到的是指针的大小而不是数组的大小

动态数组初始化

在方括号后加一个空的圆括号可以初始化动态数组. 新的标准允许提供初始化列表. 如果列表元素少于数组大小, 剩下的元素值初始化. 反之多于数组大小, 分配失败, 抛出 bad_array_new_length

允许 new 一个空数组

如果大小为 0, new 返回一个合法非空指针, 不过不能解引用.

释放动态数组

delete 后加一个空的方括号

1

delete[] pia;  //方 括号不可省略

智能指针和动态数组

unique_ptr<T[]> up 不能使用 ‘.’ 或 ‘->’. 可以使用下标运算符访问.

1
2
3

unique_ptr<int[]> up(mew int[10]);
for(int i = 0; i< 10; ++i)
    up[i] = i;

shared_ptr 不能直接管理动态数组, 如果想要使用必须提供 deleter, 且其不支持下标运算符和数学运算, 访问需要使用 get 获取内建指针

1
2

shared_ptr<int[]> sp(new int[10], [](int* p) { delete[] p; });
// shared_ptr<int[]> sp(new int[10], default_delete<int[]>())

分配器(allocator class)

new 将内存分配与构造对象相关联, 当分配一个对象时, 这通常是我们想要的.

当我们分配一块内存, 并想要按需构造时, 我们想要分离分配内存与构造对象. 这时关联分配与构造有可能是浪费的. 特别的, 没有默认构造函数时无法使用动态数组

allocator class

allocator 提供类型明确的未构造内存.

allocator<T> a 定义一个名为 a 的可以分配类型为 T 的对象的 allocator
a.allocate(n) 分配可以保存 n 个 T 对象的未构造内存
a.deallocate(p, n) 释放内存
a.constuc(p, args) 构造对象
a.destroy(p) 析构对象

allocator<string> alloc;
auto const p = alloc.allocate(n);
// 构造
auto q = p;

// 析构
while (q != p) alloc.destroy(q--);

// 释放内存
alloc.deallocte(p, n);

未初始化内存的拷贝或填充算法

uninitialized_copy_n(beg, n, beg2) 拷贝 n 个元素
uninitialized_fill(beg, end, t) beg 和 end 之间的元素拷贝到 beg2, 需要有足够空间
uninitialized_fill_n(beg, n, t) 拷贝 n 个元素
