计算顺序

算术操作符 > 比较操作符 > 逻辑操作符

使用 || 连接串

  • 连接不会加入空格
  • 可以连续操作
  • 与 NULL 相连为 NULL
  • 连接串与非串时, 如果 DBMS 不能隐式转换, 需要将非串转为串

MySQL 使用||作为 OR 的同意词. 连接串使用 CONCAT()

SUBSTRING() 提取子串

SUBSTRING(str FROM beg [FOR length])

UPPER() 和 LOWER()

转为大写和转为小写

TRIM()

删除串两端不需要的字符

  • 只能处理前缀和后缀, 不能处理中间字符. 默认修整前缀和后缀
  • 默认修整空格
  • 通常用于格式化结果和 WHERE 中的比较

TRIM([[LEADING | TRATLING| BOTH][trim_char][FROM]] string)

CHARACTR_LENGTH()

返回串中字符个数

  • 是字符的个数
  • 字节个数可以用 BIT_LENGTH(), OCTET_LENGTH(). MySQL 使用 BIT_COUNT()

POSITION 查找子串

POSITION(substring IN string)

  • REPLACE 可以替换

SELECT title_id, pubdata
  FROM titles
  WHERE EXTRACT(YEAR FROM pubdata)
          BETWEEN 2001 AND 2002
    AND EXTRACT(MONTH FROM pubdata)
          BETWEEN 1 AND 6
 ORDER BY pubdata DESC;

类型分配

使用 CAST() 转换数据类型

  • 可以将任意的数字或日期转为字符数据类型
  • 符合目标类型格式的字符类型可以转为相应类型
  • 某些数字转换会发生舍入或截断(取决于 DBMS)
  • VARCHAR 转 CHAR 会截断
  • 数字转换溢出时有可能发生错误

CAST(expr AS data_type)

NULLIF()
