顺序容器

顺序表中元素的存储顺序与添加顺序相同. STL 还提供了一些联合容器, 存储顺序根据与元素关联的 key. 容器共享了一个普遍的接口, 并用各自的方法实现.

概论

容器 表现
vector 可变长数组, 支持快速的随机访问. 在非末尾插入和删除会很慢
deque 双端队列. 支持快速的随机访问, 在头/尾可以快速插入和删除
list 双向链表. 只支持双向顺序访问. 可以在任何位置快速插入
foreword_list 单向链表. 只支持单向顺序访问. 可以在任何位置快速插入
array 定长数组. 支持随机访问. 不能插入和删除
string 专用容器, 类似保存字符类型的 vector. 随机访问, 快速末端插入和删除

foreword_list 没有 size 操作. 其它容器, size 是快速常量复杂度的操作.

选择容器

  1. vector 是首选的
  2. 需要随机访问, vector 或 deque
  3. 需要频繁在中部插入和删除, 使用 lsit 或 forward_list
  4. 头部和尾部的插入和删除, deque

库概论

几乎所有容器都支持的操作

构造器

  • 默认构造: C c;
  • 拷贝构造: C c1(c2);
  • 迭代器: C c(beginIterator, endIterator); array 不支持
  • 初始化列表: C c{a, b, ...}
  • 指定大小: C c(n)
  • 指定大小和默认值: C c(n, value)

赋值和交换

  • c1 = c2
  • c1 = {a, b, ...} array 不支持
  • a.swap(b), 交换 a,b
  • swap(a, b), 与 a.swap(b) 相同

长度

  • c.size() : c 中元素个数
  • c.max_size(): c 可以保存元素的最大数目
  • c.empty(): c 没有元素时返回 true, 否则返回 false

添加和删除元素(array 不支持)

  • c.insert(args) 拷贝 args 到 c
  • c.emplace(inits) 根据 inits 构造 c 中的元素
  • c.earse(args) 移除 args
  • c.clear() 移除所有元素

相等和关系运算

  • ==, !=: 所有容器都支持
  • <, <=, >, >=: 与元素是否可比有关

迭代器(iterator)

支持 ++ 和 –. forward_list 不支持 –. string, vector, deque 支持算术加减法

迭代区间

[begin, end)

操作

adding

  • push_back forward_list 不支持
  • push_front string, vector 不支持
  • emplace_back forward_list 不支持
  • emplace_front string, vector 不支持
  • insert(p, t)
  • insert(p, n, t)
  • emplace(p, t)
  • emplace(p, n, t)

访问

  • c[n], 越界未定义
  • c.front() 返回引用
  • c.back() 返回引用
  • c.at[n], 越界抛出 out_of_range

earseing

  • pop_back forward_list 不支持
  • pop_front string, vector 不支持
  • earse(p)
  • earse(begin, end)
  • clear

forward_list

添加和删除元素需要修改其前驱, 所以 forward_list 添加到后面

  • insert_after
  • emplace_after
  • earse_after

为了添加头, 定义了 before_begin

会使迭代器非法的操作

插入

  • vector 和 string 如果重新分配了内存, 迭代器, 指针, 引用会非法. 如果没有重新分配, 插入之前的仍然合法, 之后的不合法
  • deque 除了在开头和结尾插入, 迭代器, 指针, 引用都会非法. 在开头和结尾插入, 迭代器非法, 不过存在元素的指针和引用合法
  • list forward_list 仍然合法

适配器

三个适配器. stack, queue, priority_queue . 适配器是一个概念. 有容器, 迭代器, 函数适配器. 简单来说, 适配器是一个让某个东西的行为像另一个

定义一个适配器

  • 默认构造 stack<int> s;
  • 通过另一个容器 stack<int> s(deq); deq 是 deque 对象

stack 和 queue 默认是由 deque 实现的. priority_queue 默认是由 vector 实现的. 可以在声明时的第二个类型参数改变默认实现

1

stack<int, vector<int>> S;
