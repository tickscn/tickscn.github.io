简介

操作系统的功能

  1. 计算机系统
    • 硬件
      • 中央处理器(Central Processing Unit, CPU)
      • 内存(Memory)
      • 输入/输出设备(Input/Output device, I/O device
    • 软件
    • 数据
  2. 可将操作系统看作资源分配器

计算机系统的运行

CPU 与各设备控制器并发执行, 并且竞争访问内存.

开机

  1. 引导程序(bootstrap program)
    1. 通常位于计算机的固件(firmware), 如只读存储器(Read-Only Memory, ROM).
    2. 初始化组件, 并将内核加载到内存
  2. 内核(Kernel)
    1. 除了内核, 系统程序加载到内存成为系统进程(system process) 或系统后台程序(system daemon), 生命周期与内核一样
    2. 等待事件发生
  3. 中断(interrupt)
    1. 硬件通过系统总线发送信号到 CPU
    2. 软件通过系统调用

存储结构

CPU 从内存加载指令, 通用计算机的运行程序位于主存(main memory), 通常为动态随机访问内存(Dynamic Random Access Memory, DRAM)

内存的缺陷

  1. 容量较小
  2. 内存是易失性(volatile), 掉电失去所有数据

需要外存(secondary storage)

  1. 常用磁盘或硬盘(magnetic disk)

I/O 结构

  1. 驱动加载适当寄存器
  2. 控制器检查寄存器, 决定采取操作
  3. 加载数据到本地缓冲区
  4. 中断

操作系统的结构

  1. 分时(time shared)
  2. 调度(scheduling)

操作系统执行

  • 中断驱动(interrupt driven)

双重模式与多重模式的执行

至少需要两种单独运行模式

  1. 用户模式(user mode)
  2. 内核模式(kernel mode) ( 监视模式(supervisor mode), 系统模式(system mode), 特权模式(privileged mode))

定时器

防止用户程序一直占用, 可通过计时器来产生中断

进程管理

  • 在CPU上调度进程和线程
  • 创建和删除进程
  • 挂起和重启进程
  • 进程同步
  • 进程通信

存储管理

逻辑存储单元–文件(file)

其它

  1. 保护和安全
  2. 内核数据结构
    • 单向链表
    • 双向链表
    • 循环链表
    • 堆栈
    • 队列
    • 树
    • 哈希函数和哈希表
    • 位图
  3. 计算环境
  4. 历史

    2. BSD Unix
