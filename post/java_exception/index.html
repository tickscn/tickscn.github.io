异常

异常分类

Java 中异常对象都派生于 Throwable 类的一个实例.分为两个分支

  • Error
  • Exception

Error 类表示系统内部错误和资源耗尽错误. 只能通告用户并尽力使程序安全退出

Exception

  • RuntimeException 程序运行异常
  • 其它异常. 程序没问题如 IOException

Error 和 RuntimeException 被称为非受查异常,其它异常被称为受查异常

声明受查异常

方法需要在首部声明所有可能发生的异常

应该抛出异常的情况

  1. 调用一个抛出受查异常的方法, 例如 FileInputStream 构造器
  2. 运行中发现错误, 抛出一个受查异常
  3. 程序出现错误
  4. Java 虚拟机和运行时库出现的内部错误

对于可能被他人使用的 Java 方法, 应该在方法和首部声明所有可能抛出的受查异常

捕获异常

有些代码可以直接抛出异常不用处理, 而有写代码必须捕获异常

捕获异常

如果异常发生并且没有被捕获, 程序将终止并打印异常信息. 想要捕获异常需要 try/catch 语句块

1
2
3
4
5
6

try{
    //code
}
catch{ExceptionType e}{
    //code
}

如果 try 块抛出的异常在 catch 子句中有说明. 程序将跳转到对应 catch 块

捕获多个异常

用 | 符号分隔

可以抛出新的异常, 也可以用当前异常为原因包装新异常

使用异常的技巧

  1. 不要用异常来代替简单的测试, 如栈是否为空, 对象是否为 null
  2. 不要过分细化异常
  3. 利用异常层次结构
  4. 不要压制异常
  5. 早抛出, 晚捕获

使用断言

assert

assert condition;

assert condition: expression;

如果 condition 为 false 会抛出 AssertionError 异常, 第二中情况表达式会被传入异常的构造器

1

c/c++ 中 assert 是宏, 表达式是 false 时会打印表达式

启用和禁用

默认情况断言被禁用, 可以在运行程序时使用 -enableassertions 或 -ea 开启断言, 可以 -ea:… 启用特定包的断言, 也可以用 -disableassertions:… 或 -da:… 禁用特定包或类的断言

日志

调试时, 有时会使用打印的方法来判断. 解决问题后这些打印的代码应该去除. 使用日志 API 可以有更好的效果

  • 可以很容易的取消或打开日志记录
  • 可以简单的禁用日志输出
  • 可以被定向到不同的处理器, 如控制台标准输出, 日志文件
  • 可以过滤日志
  • 可以用不同的方法格式化
  • 默认情况由配置文件控制

基本日志

全局日志记录器 Logger.getGlobal().info();

高级日志

企业级(industrial-strength)日志, 自定义日志记录器

1

private final static Logger mylogger = Logger.getLogger("com.mycompany.myapp");

日志记录器具有层次结构. 子记录器会继承父记录器的一些属性

日志记录级别

  • SEVSRE
  • WARNING
  • INFO
  • CONFIG
  • FINE
  • FINER
  • FINEST

默认只记录前三个级别, 也可设置其它级别, 如 logger.setLevel(Level.FINE) 开启 FINE 和更高的级别, Level.ALL 开启所有. Level.OFF 关闭所有

记录方法

  • logger.waring(message)
  • logger.fine(message)

也可使用 log 方法指定级别

  • logger.log(Level.WARING, message)

修改日志管理器配置

默认配置文件 jre/lib/logging.properties 更改配置文件需要设置 java.util.logging.config.file 设置为配置文件存储目录, 启动命令

1

java -Djava.util.logging.config.file=configFile MainClass

可以重新初始化

本地化

完成本地化需要资源包(resource bundle)中的本地特信息. 包含字符串的映射. 资源包的名字 com.company.logmessages

  • com.company.logmessages_en.properties
  • com.company.logmessages_cn.properties

资源包是纯文本文件. 格式为

  • key=direc

getLogger 方法可以指定资源包. 记录的时候 message 为资源包中的关键字

处理器

默认控置台处理器 ConsoleHandler. 默认级别为 INFO. 可以通过配置文件更改, 也可以使用自己的记录器与处理器

myhandler.setLevel(Level.FINE);
mylogger.addHandler(myhandler);
