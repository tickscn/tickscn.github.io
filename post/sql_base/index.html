SQL 语法

  • 注释: 以两个连字符开始 –, 持续到行尾
  • SQL 语句: 关键字引入的标记的的有效组合, 包括关键字, 标识符, 操作符, 字面量和标点
  • 子句: SQL 语句片断
  • 关键字: SQL 保留字
  • 标识符: 用于数据库对象命名的词
  • 分号: 语句以分号 ‘;’ 结尾

SQL 是形式自由的语言, 它的语句

  • 可以是大写或小写字母
  • 只要不把单词, 标记, 或引号字符串分为两部分, 可以在下一行继续
  • 和其它语句同一行
  • 从任一列开始

不过建议采用一致的风格, 使用大写关键字和小写标识符

SQL 标准和一致性

SQL 标准复杂, 不同的 DBMS 支持 SQL 不同版本的不同级别. 号称符合标准 SQL 需要遵循 Core SQL 的标准子集

标识符

用来引用一个对象的名称, 在它的作用域内需是唯一的. 标准 SQL 命名规则

  • 128 个字符以内
  • 以字母开头
  • 可以包含字母, 数字, 下划线(_)
  • 不可以包含特殊字符(如 # $ @ & % 和标点符号)
  • 不可以是保留关键字(带引号的标识符可以绕过一些限制, 不过不建议)

Exact numeric 精确数字

字符串类型

  • 0 或多个字符的有序序列
  • 长度可以是固定或可变
  • 区分大小写
  • 被 单引号 环绕
  • 可以是
类型 描述
CHARACTER/CHAR CHARACTER(n) 最大长度是 n 的字符串, n >= 1. 最大长度依赖 DBMS
CHARACTER VARYING/ CHAR VARYING/ VARCHAR VARCHAR(n) 最大长度是 n 的字符串, 当长度小于 n 时不以空格填充, 而是只存该字字符串
NATIONAL CHARACTER/NCHAR/NATIONAL CHAR 保存标准多字节符或 unicode
NATIONAL CHARACTER VARYING
CLOB 大型字符对象, 无法用索引, 支持较少的操作符和函数. CHARACTER LARGE OBJECT 同义词
NCLOB NATIONAL CHARACTER LARGE OBJECT

二进制大型对象类型

Binary large object, BLOB 类型

  • 类型名: BLOB 或 BINARY LARGE OBJECT
  • 存储长的字节序列
  • 存储大数据量的二进制数据(图形, 照片, 音频, 视频等)
  • 不能用作关键字或索引

布尔类型

  • 类型名 BOOLEAN
  • 字面量 TRUE, FALSE, UNKNOWN
  • NULL 等价于 UNKNOWN, 通常被用来替代 UNKNOWN

日期和时间

  • 依据世界协调时(UTC)来定义的, SQL 标准要求每个 SQL 会话有一个默认时区
  • 公历(格里历, gregorian calender)
  • 时间采用 24 小时制
  • 连字符(-)连接日期, 冒号(:)连接时间. 空格连接日期和时间
类型 说明
DATA 由三个整数段(YEAR, MONTH, DAY)组成, 格式化 yyyy-mm-dd, 如 2020-06-11
TIME 由三个整数段(HOUR, MINUTE, SECOND)组成, 格式化 hh:mm:ss , 如 19:33:44
TIMESTAMP 以空格分隔的 DATA 和 TIME. 如(2020-06-11 19:33:44)
TIME WITH TIMEZONE 由 TIME 添加一个表示时区的段
TIMESTAMP WITH TIMEZONE

有效值

YEAR 0001-9999
MONTH 01~12
DAY 01~31
HOUR 00~23
MINUTE 00~59
SECOND 00~61.999…
TIME_ZONE_OFFSET -12:59 ~ +13:00
  • 为了插入闰秒, 秒的取值范围可以到 61.999…
  • 字面量 DATA' yyyy-mm-dd', TIME' hh:mm:ss', TIMESTAMP' yyyy-mm-dd hh:mm:ss'
  • 标准 SQL 不处理公元前

时间间格类型

  • 存储两个时间戳的差
  • 可以用来增加或减少日期和时间
YEAR-MONTH INTERVAL ‘15-3’
DAY-TIME INTERVAL ‘20:05:33’
