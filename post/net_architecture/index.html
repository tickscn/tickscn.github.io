典型计算机网络体系结构

OSI/RM

开放系统互连参考模型(Open System Interconnection Reference Model, OSI/RM) 是第一个标准化的计算机网络体系结构, 分为 7 层

  1. 应用层(Application Layer) ->提供网络应用接口
  2. 表示层(Presentation Layer) -> 数据表示, 压缩, 加密
  3. 会话层(Session Layer) -> 主机间的会话管理
  4. 传输层(Transport Layer) ->端到端的可靠传输
  5. 网络层(Network Layer) ->寻址主路由选择
  6. 数据链路层(Data Link Layer) ->相邻节点间可靠传输
  7. 物理层(Physical Layer) -> 比特流传输

划分有以下原则

  • 同一层中的网络节点具有相同的层次结构, 具有同样的功能
  • 同一节点内相邻层之间通过接口通信
  • 使用下一层提供的服务, 并为上一层提供服务
  • 不同节点的相同层按照协议实现对等层通信

TCP/IP

划分了四层

  1. 应用层
  2. 传输层
  3. 网际互联层(Internet Layer)
  4. 网络访问层(Network Access Layer)

是针对 TCP/IP 协议簇的体系结构

TCP/IP 4 层结构 建议的 TCP/IP 5 层结构 OSI/RM
应用层 应用层 应用层
表示层

网络访问层 数据链路层 数据链路层

局域网体系结构

有线局域网(IEEE 802.1A)

  1. 数据链路层
    1. LLC 子层
    2. MAC 子层
  2. 物理层

局域网中不需要路由寻址, 传输层和应用层由操作系统中的通信协议和应用软件实现. 所以只需要低层. 而且局域网中易出现介质争用现象, 将数据链路层细分为介质访问控制(Media Access Control, MAC)层和逻辑链路控制(Logical Link Control, LLC)层

WLAN

结构相似, 但是协议和介质不同

网络体系对等通信原理

因为只有对等层有相同类型的协议可以相互通信

PDU(Protocol Data Unit, 协议数据单元)

  • 物理层 PDU 是比特流
  • 数据链路层是帧(frame). 包含多个比特
  • 网络层的传输单位是包(paket)
  • 传输层 OSI/RM TPDU. TCP/IP 数据段(segment). UDP 数据报(datagram)

通信协议

通信协议是通信时的约定和约束, 有三要素

语义

描述协议的功能和版本

语法

规定信息的格式, 包括数据和控制信息的格式, 编码, 电平信息

同步

规定行为次序
