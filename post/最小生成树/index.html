最小生成树

2019-11-26
算法导论

文章目录

  • 算法思路
    • 辨认安全边
  • Kruskal 算法
  • Prim 算法

一个边通无向图 G = (V, E), 为每一条边(u, v)$\in$ E, 赋予权重 w(u, v), 那么 G 的一个无环子集 , 使得 T 连通所有结点且权重最小. 该子集 T 被称作 最小生成树. 求取 T 的问题为最小生成树问题

算法思路

生成一个边集 A, 满足 A 是最小生成树 T 的子集. 那么循环的每一步就是选择一条边(u, v) $\in$ E, 且$A\cup {(u, v)}\subseteq T$, 这样的边暂时称为安全的边. 算法的关键是选择安全的边, 有两种出名的贪心算法来解决这个问题

辨认安全边

无向图 G(V, E)的切割(S, V-S)是一个划分. 若边 (u, v)$\in$E, 且 u$in$S, V$in$V-S 或反过来. 则称边(u, v)橫跨切割. 如果切割没有分开集合 A, 即 A 中没有橫跨切割的边, 称切割尊重集合 A.橫跨切割的边中权重最小的边称为轻量级边.

设 G = (V, E)是带权连通无向图, A 是它某一最小生成树的子集. 切割(S, V-S)尊重 A, 那么(S, V-S)的一条轻量级边 对 A 是安全的

可用剪切-粘贴法来证明

Kruskal 算法

  1. 初始化 A 为空集, |V|个树, 每个树包含一个结点.
  2. 根据权重对边排序
  3. 根据权重从小到大, 若边(u, v)的端点 u, v 属于不同的树, A = A$\cup$ {(u, v)}. 合并树 u, v

MST-KRUSKAL(G, w)
    A = emptyset
    for each vertex v in G.V
        MAKE-SET(v)
    sort the edges of G.E into nondecreasing order by weight
    for each dege(u, v) in G.E
        if FIND-SET(u) != FIND-SET(v)
            A += {(u,v)}
            UNION(u, v)
    return A

MST-PRIM(G, w, r)
    for each u in G.V
        u.key = inf
        u.pre = NIL
    r.key = 0
    Q(G.V) //< 最小优先队列
    while Q is not empty
        u = EXTRACT-MIN(Q)
        for each v in G.Adj[u]
            if v in Q and w(u,v) < v.key
                v.key = w(u, v)
                v.pre = u
