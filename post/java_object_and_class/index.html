类

对象

对象的主要特征

  1. 行为. 允许的操作, 类提供的方法
  2. 状态.
  3. 标识

用户自定义类

构造器

同 C++ 类似, 不过总是伴随 new 操作符

隐式参数和显式参数

  1. 隐式参数,是方法的调用者. this
  2. 显式参数是括号中明确的参数
1

Java 中方法都定义在类中

final

1

Java 中 =final= 建议修饰基本类型或不可变域的类

静态域和静态方法

静态域

被 static 修饰. 类的所有对象共享同一个域.

1

静态域又称为类域

静态方法

不能向对象实施操作的方法

1

可以使用对象调用静态方法, 不过建议使用类名来调用
  1. 不需要隐式参数
  2. 只访问静态参数

工厂方法(factory method)

是静态方法的另一用途. 以后会详细说明

  1. 实例采用不同的构造名字, 构造器无法实现
  2. 构造器无法改变构造对象类型

main 方法

main 方法启动时没有具体的对象.

1
2

不同的类可以有自己的 main 方法, 只需要在 Java 虚拟机启动时指定启动的类.
可以用来进行测试

方法参数

Java 总是采取按值调用

  1. 不能修改基本类型参数
  2. 可以改变对象参数的状态
  3. 不能对象参数引用一个新的对象

对象构造

重载

方法的签名包括方法名, 参数. 不包含返回值

默认域初始化

与局部变量不同, 域的值会默认初始化, 数据类型是 0, boolean 是 false, 对象引用是 null

显式域初始化

可以在类定义中将任何值赋值给域

调用另一个构造器

通过 this(...) 调用另一个构造器

初始化块

类的声明中可以包含多个代码块, 会在对象实例化的时候执行初始化静态类型的代码块可以用 static 标记

包

用来组织类, 类似于 namespace

导入类

  1. 使用完整的包名
  2. import . 使用一个 * 导入所有类. 明确导入的类有时可以提高可读性

静态导入

import 不仅可以导入类, 也可以导入静态方法和静态域

将类放入包中

  • 在文件开头 package 包名
  • 将文件放入与包名相对应的目录中

包作用域

  • public 任意类都可使用
  • private 本类使用
  • 没有指定时可被同一个包的所有方法访问

类路径

类存储在子目录中. 路径需要与包名匹配

  1. 把类放入一个目录中. 如 ~/classdir
  2. 把 JAR 文件放入另一个目录中, 如 ~/javaarchives
  3. 设置类路径
    • *unix 使用冒号分隔(~/classdir:.:~/javaarchives)
    • Windows 使用分号分隔 (c:\classdir;.;c:\javaarchives)

设置类路径

  1. java 使用 -classpath 或者 -cp 指定类路径
  2. 设置环境变量 CLASSPATH

文档注释

以 /** 开始的注释可以被 javadoc 生成 html 文档

注释的插入

javadoc 从下面几个方面抽取文档

  • 包
  • 公有类与接口
  • 公有的和受保护的构造器与方法
  • 公有的和受保护的域

类注释

import 之后, 类之前

2
3
4
5

/**
 * A {@code Student} object.
 */
public class Student{
    //...
}

方法注释

放在方法之前

  • @param 参数描述
  • @return 返回值描述
  • @throws 异常描述

域注释

通常只需要对公有域进行注释

通用注释

  1. 类文档
    • @author 作者
    • @version 版本
  2. 所有文档
    • @since 始于…, 可以用来标记引入某个特性的版本
    • @deprecated 对不在使用的类, 方法, 域提示
    • @see 超链接 @see package.class#label

包注释

  1. package.html 中 … 之间的文本
  2. package-info.java 包语句之后唯一的文档注释

类设计技巧

  1. 数据私有
  2. 初始化
  3. 减少基本类型. 把相关的数据抽象为一个新的类
  4. 按需实现访问器和修改器
  5. 不要让一个类过于复杂
  6. 类名和方法名应有意义, 最好从名子可以看出作用
  7. 如果可以, 使用不可变类
