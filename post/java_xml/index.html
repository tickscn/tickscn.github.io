XML 概述

XML 与 HTML 都继承于标准通用标记语言(Standard Generalized Markup Language, SGML), 两者有很大的相似性.

XML 文档结构

文档头

1
2

<?xml version="1.0"?>
<?xml version="1.0" encoding="UTF-8"?>

文档类型定义

在文档头后通常是文档类型定义(Document Type Definition, DTD), 用来确保文档正确, 非必须

1

<!DOCTYPE xxx ...>

元素

元素可以包含子元素和文本, 还可以有属性

1
2
3
4

<font unit="pt">
    <name>Hei</name>
    <size>30</size>
</font>

1

如果避免混合式内容, 可以简化解析过程

有用的标记

  • 字符引用: &# 十进制;或&#x 十六进制. 例 é 可以用 &#233, &#e9 表示
  • 实体引用
    • 引号: &quot
    • 省略号: &apos
    • &: &amp
    • 小于: &lt
    • 大于: &gt
    • 在 DTD 中定义的其它实体引用
  • CDATA 用来包含特殊符号, <![CDATA[…]]>
  • 处理指令
  • 注释

解析 XML 文档

  • 文档对模型(Document Object Model, DOM), 将文档解析成树

验证 XML 文档

如果指定文档结构, 解析时就可以更加简单. 可以使用 DTD 或 schema 来指定文档结构. 如一项 DTD 规则

<!ELEMENT font (name, size)>

表明一个 font 元素必须是两个子元素, 分别是 name 和 size. SML schema

1
2
3
4
5
6

<xsd:element name="font">
    <xsd:squeuence>
        <xsd:element name="name" type="xsd:string"/>
        <xsd:element name="size" type="xsd:int"/>
    </xsd:squeuence>
</xsd:element>

DTD

1

<!DOCTYPE configuration []>

[]中写 DTD 规则. 不过会使得文件过长, 常用的是将 DTD 规则写入文件. 使用 SYSTEM 声明

1

<!DOCTYPE configuration SYSTEM "config.dtd">

ELEMENT

用于指定某个元素的子元素

规则 含义
E* 0 或多个 E
E+ 1 或多个 E
E? 0 或 1 个 E
E1| E2| …| En 其中的一个
#PCDATA 文本
ANY
EMPTY

ATTLIST

描述合法元素属性

类型 含义
CDATA 任意字符串
(xxx| yyyy| …) 属性之一
ID 唯一 ID
IDREF, IDREFS 对 ID 的引用
NMTOKEN, NMTOKENS 名字标记
ENTITY, RNTITYS 未解析实体

默认值

默认值 含义
#REQUIRED 必须
#IMPLIED 可选
A 可选, 默认是 A
#FIXED A 未指定或 A, 结果都是 A

使用 XPath 来定位信息

#begin_src … … xxx … … … #end_src 对 XPath 表达式/configure/database/username 求值可以获得 username 的值

  1. 创建 XPath 对象 #begin_src XPathFactory xpfactory = XPathFactory.newInstance(); XPath xpath = xpfactory.newXPath(); #end_src
  2. 调用 evaluate 计算该表达式.
    1. String username = xpath.evaluate("/configuration/database/username", doc);
    2. NodeList nodes = xpath.evaluate("/configuration/database", doc, XPathContants.NodeList)
    3. Node node = xpath.evaluate("/configuration/database[1]", doc, XPathContants.Node)
    4. 数字对应修改

使用命名空间

XML 的命名空间(namespace)可以用于元素名和属性名. 名命空间由统一资源标识符(Uniform Resource Identifier, URI)来标识. #begin_example http://xxxx uuid:xxxxxx urn:com:xxx #end_example 命名空间使用统一资源标识符(Uniform Resource Identifier, URI)
