函数

  • 函数定义(function definition)包括返回类型, 函数名, 包含 0 或多个的变量列表, 函数体
  • 使用前必须声明: 函数定义中的函数体被分号代替, 可以省略参数名

数组参数

数组参数的特殊

  • 不能按值传递
  • 通常被转为指针

  3. 传递一个大小参数

使用引用

1

void print(int (&arr)[10]);  //OK, arr 是有 10 个 int 元素的数组的引用

因为数组大小是数组类型的一部分, 上面的声明方式导致 print 只能用于含有 10 个元素的数组

可变参数

有时不能提前确定参数的个数, 新的标准支持两和主要的方式来实现可变数量的参数

初始化列表类型(initializer-list)

如果函数的参数类型都相同, 可以使用 initializer-list 类型. 需要头文件 #include<initializer-list> 该类型支持的操作

initializer-list lst 默认构造函数, 空的列表
initializer-list lst {a,b,c,…} 初始化, 列表中元素是 const 的
lst2(lst) copy 构造, 不复制元素
lst2 = lst copy 构造或赋值, 不复制元素, 两者元素共享
lst.size() 大小
list.begin()
list.end()

省略号参数 ...

省略号参数(elipsis parameters)是为了兼容 C 语言的

inline 和 constexpr

inline 只是对编译器的 建议, 编译器可以选择忽略这个关键字

constexpr 函数

constexpr 声明的函数是编译期函数, 可以当作常量表达式, 隐式指定为内联(inline)的

  1. 参数和返回值类型必须是字面量(literal type)
  2. 可以递归
  3. 不能是虚函数
  4. 不能包含 goto 或 try 块
  5. 可能返回非常量

建议在头文件中定义 inline 和 constexpr 函数

NDEBUG

预处理器变量 NDEBUG 控制 assert 的行为. 如果定义了 NDEBUG, assert 就被忽略

  1. 可以使用 #define NDEBUG
  2. 使用命令行参数 CC -DNDEBUG ...

编译器定义的常用变量

  1. __func__: 函数名
  2. __FILE__: 文件名
  3. __LINE__: 当前行号
  4. __TIME__: 编译时间
  5. __DATE__: 编译日期

函数匹配

可选函数(candidate function)

名字相同且在当前空间可见的函数.

可实施函数(viable function)

拥有相同的参数数目, 且参数类型相同或可以转换

找到最优匹配

函数指针

定义一个指针

函数指针需要特定的函数类型, 函数类型由返回值类型与参数类型组成. 如 bool stringEqual(const string&, const string&) 的类型是 bool (const string&, const string&)

bool (*pf)(cosnt string&, const string&) 一个可以指向 stringEqual 的未初始化函数指针

() 不能省略, 不然定义的是返回指针的普通函数

使用

函数指针的赋值, 同时支持以不两种

  1. pf=stringEqual
  2. pf=&stringEqual

使用

  1. pf("abc", "abd")
  2. (*pf)("abc", "abd")

函数指针作为函数参数
