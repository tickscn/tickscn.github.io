DHCP

动态主机配置协议(DHCP)一种用来简化 IP 配置的管理的服务. 可以为安装了 DHCP 客户端的主机自动配置动态 IP

DHCP 服务维户一个 IP 地址池, 其中保存着可用的 IP 地址, 再需要的时候可以分配出去

主要功能和应用环境

主要功能

  1. 基本的 IP 分配
  2. 通过 IP 与 MAC 绑定实现静态 IP 分配
  3. 配置客户端的 DNS 服务器, 默认网关, Windows 还可配置 WINS 服务器
  4. 利用 IP 地址排除, 不在将以经静态分配给其它主机的 IP 地址再分配
  5. 利用 DHCP 中继功能为不同网段的中的 DHCP 客户端分配不同地址池中的 IP 地址

优点

  1. 简化配置和管理
  2. 方便客户端移动
  3. 配置更加可靠

应用环境

  • 大规模的网络
  • 用户数目大于 IP 地址空间, 只能使用动态分配来满足需要
  • 当网络中只有少数主机需要固定的 IP 地址

DHCP 报文与格式

与 DNS 一样是 C/S 模式, 不过客户端使用 UDP 68 发送请求报文, DHCP 服务端用 UDP 67 发送应答报文

DHCP 报文类型

  1. DHCP DISCOVER: 以广播方式发送该报文, 收到消息的 DHCP 服务器会都会应答, 客户端由此得知网络中的 DHCP 服务器地址
  2. DHCP OFFER: 收到 DISCOVER 报文的 DHCP 服务器会选择一个 IP 地址, 加上租约期限, DNS 服务器, 网关等信息构造一个 OFFER 报文, 发送给 DHCP 客户端, 告知客户端本服务器可以提供相应服务
  3. DHCP REQUEST: 如果客户端收到多个服务器发来的 OFFER 报文, 需要选择一个(通常为最早收到的报文)的发送端作为目标服务器, 并向其发送 REQUEST 报文, 通知它希望获得所分配的 IP 地址. 另外当租约过去一半时也会发送 REQUEST 报文请求续约, 如果没收到确认会在只剩下 1/4 的时候再次发送 REQUEST 请求续约
  4. DHCP ACK: 收到客户端的 REQUEST 报文后, 查看 MAC 地址是否有租约记录, 如果有就发送 ACK 报文, 通知客户端可以使用分配的 IP 地址
  5. DHCP NACK: 无租约记录或无法分配 IP 地址, 通知客户端无法分配 IP
  6. DHCP RELEASE: 当 DHCP 客户端不再需要使用分配的 IP 地址时, 就会向 DHCP 服务器发送 RELEASE 报文请求释放 IP 地址
  7. DHCP DECLINE: 当客户端通过地址冲突检查发现分配的地址不可用时, 会用 DECLINE 报文通知服务器
  8. DHCP INFORM: 目前以基本不用了, 是客户端为了向服务器查询更多的配置信息时使用

DHCP 报文格式

  1. OP: 指明报文类型(8 位). 1 是请求, 2 是应答
  2. Htype, Hlen.(各 8 位) 客户端的 MAC 地址类型和长度
  3. Hops:(8 位) DHCP 中继计数
  4. Xid: (32 位) 一个用来标识报文的随机数
  5. Secs:(16 位) IP 地址被使用时间
  6. Flags: 标志位(16 位) 第一位是 0 表示单播, 1 表示广播. 其余保留
  7. Ciaddr: 客户端 IP 地址(32 位), 仅 ACK 应答报文有效, 其它情况置 0
  8. Yiaddr: OFFER 报文中为客户端分配的地址(32 位), 其它报文置 0
  9. Siaddr: 下一个提供 DHCP 服务的服务器地址(32 位), 仅 ACK 和 OFFER 报文有效
  10. Giaddr: 第一个中继的 IP 地址(32 位)
  11. Chaddr: 客户端 MAC 地址(128 位)
  12. Sname: 指示为 DHCP 客户端分配 IP 地址的 DHCP 服务器名称(512 位)
  13. File: 启动配置文件名称和路径(1024 位)
  14. 选项
    • 报文类型
    • 有效租约
    • 续约时间
    • 子网掩码
    • DNS
    • 网关
