异常捕获

抛出异常

C++ 中异常通过抛出一个表达式来产生. 表达式类型和调用链一起决定处理器. 处理器是调用链中与抛出对象类型匹配的最近的那一个, 对象的类型和内容可以告诉处理器哪里发生了问题.

当 throw 被执行时, 其后面的语句将不被执行, 程序被 catch 接管. catch 有可能与 throw 位于同一函数内部, 也有可能位于直接或间接调用该函数的函数体内. 执行跳转有两个重大的影响

  1. 调用链中的函数可能提前结束
  2. 当进入处理器时, 调用链中的对象将被销毁

栈展开

当抛出一个异常时, 当前函数被挂起, 开始查找 catch. 如果 throw 本身在 try 块中, 异常被相关的匹配 catch 捕获. 如果没有匹配的处理器, 检查上层 try 块的处理器. 如果没有匹配, 退出当前函数, 销毁局部变量, 到上层调用中查找.

如果一个匹配的 catch 被发现, 程序执行 catch 中语句, catch 完成后, 接着执行之后的语句

1

未捕获的异常会使程序中断

栈展开时局部变量自动销毁

析构函数与异常

1

析构函数不应抛出异常

异常对象

编译器使用抛出表达式拷贝初始化一个异常对象(exception object). throw 中的表达式必须有完成的类型. 特别的该类型是类的时候, 类必须有可访问的析构函数, 拷贝或移动构造函数. 如果表达式类型是数组或函数, 被转为相关的指针.

通常抛出指针类型有可能导致 bug, 这通常是错误的, 非常不建议使用.

捕获异常

catch 子名中声明的异常类似于只有一个参数的函数参数列表. 类型需是完成的, 可以是左值引用, 不能是右值引用

1

通常 catch 捕获一个依赖继承的异常时通过引用

查找匹配的处理器

不一定是最好匹配, 而是最近的可匹配项. 允许的转换

  1. 非常量转为常量
  2. 衍生类转不基类
  3. 数组转为指针; 函数被转为指针

重新抛出(rethrow)

有时一个单独的 catch 不能完全解决一个异常, 需要被调用链更上层外理, 可以重新抛出这个异常, 使用单独的 throw 语句. 只有异常被按引用传递时, 重新抛出的异常会保持已发生的改变

noexcept

帮助用户甚至编译器知道函数不会抛出任何异常. noexcept 需要在声明和定义中都有, 函数指针的声明和定义也可以有. 不应该出现在 typedef 和 type alias, (const)?(&|&&)?(noexcept)?

改变 noexcept

编译器不在编译时检查 noexcept, 它不确保函数不会抛出异常. 如果事实上抛出了异常, 程序终止

noexcept 参数

可以加一个可以转为 bool 的参数. noexcept 操作符可以返回一个右值 bool 来表明给定的表达式是否可能抛出异常.

虚继承

多重继承有可能造成对一个基类的多次继承

默认时衍生对象有基类的相互独立的子部分, 如果多次继承一个基类, 会有多个基类数据. 有时这不符合我们的期望. C++ 中可以用虚继承(virtual inheritance)解决这个问题, 虚继承让一个类分享它的基类, 被分享的基类被称为虚基类(virtual base class)

使用虚基类

在继承列表中添加关键字 virtual

1
2
3
4
5

class A {};
class B : public virtual A {};
class C : virtual public A {};

class D : public B, public C {};

支持转为基类引用和基类指针
