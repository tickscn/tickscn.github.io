原则

  1. 需要析构函数的类也需要拷贝构造和拷贝赋值
  2. 拷贝构造和拷贝赋值如果需要总是成对

\equal default

使用 \equal default 可以让编译器生成默认函数. 可以向普通函数定义一样

阻止拷贝

有些类的拷贝和赋值意义不明, 应该避免出现.

定义删除的函数 \equal delete

1
2

4

struct NoCopy{
    NoCopy() = default;
    NoCopy(NoCopy&) = delete;
    NoCopy& operator=(NoCopy&) = delete;
};

1

必须在最先声明的地方. =\equal delete= 可以用在仍何函数上. 但是请不要用在析构函数上

有时编译器生成的会是 deleted 函数

  1. 如果有成员变量的析构函数是 deleted 或不可访问的(private), 会生成 deleted 析构函数, 拷贝构造和拷贝赋值
  2. 如果有成员变量的拷贝构造是 deleted 或不可访问的, 会生成 deleted 拷贝构造
  3. 如果有成员变量的拷贝赋值是 deleted 或不可访问的, 或者存在 const 或引用变量, 会生成 deleted 拷贝构造

template <typename T>
class LinkList {
private:
    struct ListNode {
        T val;
        ListNode* next;
        ListNode(T _val) : val(_val), next(nullptr) {}
    };
    ListNode* head;
    void copy(ListNode* node)
    {
        head           = node;
        node           = node->next;
        ListNode* tail = head;
        while (node)
            {

                tail->next = new ListNode(node->val);
                tail       = tail->next;
                node       = node->next;
            }
    }
    void destroy()
    {
        while (head)
            {
                ListNode* pt = head;
                head         = head->next;
                delete pt;
            }
    }

public:
    LinkList() : head(nullptr) {}
    LinkList(const LinkList& rhl)
    {
        if (rhl.head) copy(rhl.head);
    }
    LinkList<T>& operator=(const LinkList& rhl)
    {
        if(this != &rhl)
            {
                destroy_();
                if(rhl.head) copy_(rhl.head);
            }
        return *this;
    }
    ~LinkList()
    {
        destroy_();
    }
    // ...
};

class MyString {
private:
    std::allocator<char> alloc;
    char* elements;
    char* first_free;
    char* cap;
    std::pair<char*, char*> alloc_copy(const char*, const char*);
    void chk_n_alloc()
    {
        if (size() == capacity()) reallocate();
    }
    void free();
    void reallocate();

public:
    MyString() : elements(nullptr), first_free(nullptr), cap(nullptr) {}
    MyString(const MyString&);
    MyString& operator=(const MyString&);
    size_t size()
    {
        return first_free - elements;
    }
    size_t capacity()
    {
        return cap - elements;
    }

    void push_back(char c);
    char* begin() const
    {
        return elements;
    }
    char* end() const
    {
        return first_free;
    }
    virtual ~MyString();
};

std::pair<char*, char*> MyString::alloc_copy(const char* b, const char* e)
{
    auto data = alloc.allocate(b - e);
    return {data, uninitialized_copy(b, e, data)};
}

MyString::MyString(const MyString& s)
{
    auto data = alloc_copy(s.begin(), s.end());
    elements  = data.first;
    cap = first_free = data.second;
}

MyString& MyString::operator=(const MyString& rhs)
{
    if (this != &rhs)
    {
        free();
        auto data = alloc_copy(rhs.begin(), rhs.end());
        elements  = data.first;
        cap = first_free = data.second;
    }
    return *this;
}

void MyString::free()
{
    while (first_free != elements) { alloc.destroy(--first_free); }
    alloc.deallocate(elements, capacity());
}

MyString::~MyString()
{
    free();
}

void MyString::push_back(char c)
{
    chk_n_alloc();
    alloc.construct(first_free++, c);
}

复杂对象 reallocate 时使用移动, 而不是拷贝

移动构造器和 std::move

移动对象

有时对象在被拷贝后会立即析构, 些时用移动代替拷贝可以有更好的表现

右值引用

为了支持移动操作, 定义了另一种引用类型——右值引用. 右值引用是必须引用右值的引用. 声明右值引用使用 && 替代普通引用的 &. 右值引用有一个重要特性是只能引用将要销毁的对象. 这样可以简单的移动资源的所有权.

移动构造和移动赋值

为了移动操作, 需要定义自己的移动构造函数和移动赋值函数.

移动构造函数的第一个参数是 class type 的右值引用, 并且其它参数(如果有的话)必须有默认值

1
2
3

StrVec::StrVec(StrVec&& s) noexcept : elements(s.elements), first_free(s.first_free), cap(s.cap)
{
    s.elements = s.first_free = s.cap = nullptr;
}

移动操作和库容器和异常

移动不构造新对象, 所以移动通常不会抛出异常. 如果我们写的移动操作不会抛出异常, 应该显式的告诉库, 不然它会作多余的操作来满足可能抛出的异常. 一种通知方法是使用关键字 noexcept. 如果成员函数在类外定义, 必须同时在声明和定义的地方加上 noexcept

  1. 虽然移动操作通常不会抛出异常, 仍需要确保它不会
  2. 库容器提供了异常发生时的保证操作. 例如 vector 保证 push_back 发生异常时保持原值不变
    • 重新分配存储空间时, 若使用移动构造转移元素且操作未完成时发生异常, vector 无法保证原值不变, 而拷贝构造转移可以, 只需要在发生异常时释放新分配的空间就可以了. 所以若不保证移动构造不抛异常, vector 会使用拷贝替代移动

StrVec& StrVec::operator=(StrVec&& rhs) noexcept
{
    if (this != &rhs)
    {
        free();
        elements   = rhs.elements;
        first_free = rhs.first_free;
        cap        = rhs.cap;
        rhs.elements = rhs.first_free = rhs.cap = nullptr;
    }
    return *this;
}

移动来源必须保证是可析构

移动操作并没有析构被移动的对象, 该对象之后有可能会被析构, 所以移动操作需要保证其可被析构. 如 StrVec 将原指针设为 nullptr
