lambda 捕获表达式

C++ 11 中可以在 [] 中表明 lambda 可以访问哪些本地变量, 并指定传递方式是引用或值. C++ 14 中捕获表达式还可以有初值, 可以在捕获列表中定义新的变量, 不过新变量需要初始值. 编译器根据是否有初值来判断是新变量还是捕获变量

#include <cmath>
#include <iostream>
using namespace std;
int main()
{
    int a = 42;
    cout << &a << endl;
    auto f1 = [a]() {// 按值传递, 捕获a
        cout << a - 2 << endl;
        cout << &a << endl;
    };
    auto f2 = [&a]() {// 按引用传递, 捕获a
        cout << a - 2 << endl;
        cout << &a << endl;
    };
    auto f3 = [a = a - 2]() {// "=" 右边的a是捕获的a, 左边的a是新定义的变量, 并在lambda中遮蔽原a
        cout << a - 2 << endl;
        cout << &a << endl;
    };
    f1(a);
    f2(a);
    f3(a);
    cout << a << endl;
    return 0;
}

使用这个特性, 可以将变量 move 或 forward 传入 lambda

const int i        = 42;  // i是常量
auto ai            = i;   // ai 不是常量
decltype(auto) dai = i;   // dai 是常量
i++;                      // error;
ai++;                     // OK;

int y   = 0;
int& ry = y;
++ry;  // y也会变为1
auto ay = ry;
++ay;  // y仍然是 1
decltype(auto) day = ry;
++day;  // y 变为 2

constexpr 要求放宽

C++ 11 中 constexpr 指定的函数只能包含有限的语法集(typedef, using, 和单独的返回语句) C++ 14 中 constexpr 允许更多的语法集, 如分支, 循环

using namespace std;
constexpr int f(int i)
{
    if (i > 42) return 43;
    return 42;
}
int main()
{
    int a = f(67);
    return 0;
}

          // warring：‘void f()’ is deprecated [-Wdeprecated-declarations]

std::make_unique

用来生成 unique_ptr 的实例的一种推荐方法. 推荐的原因有

  • 避免使用 new
  • 指定指针指向的类型时避免代码重复
  • 异常安全, 假设如下方法调用一个函数
1

foo(std::unique_ptr<T>{new T{}}, functions_may_throw(), std::unique_ptr<T>{new T{}});

在执行函数体时, 参数必须先完成运算, 但是 C++ 没有规定参数的运算顺序, 当顺序为 new T{}, functions_may_throw() … unique_ptr 构造时, 有可能在构造智能指针之前抛出异常, 造成内存泄露
