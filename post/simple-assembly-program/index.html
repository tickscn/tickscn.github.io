一个简单的汇编程序

CPUID 指令

根据EAX寄存器的值, CPUID 指令将信息存入 EBX, ECX, EDX

EAX Value CPUID Output
0 销售商 ID string, 和最大支持的选项值
1 处理器型号, 家族,模型,平台信息
2 处理器缓存设置
3 处理器序列号
4 缓存设置(线程数, 核心数, 物理性能)
5 显示器信息
80000000h 扩展销售商 ID string, 和支持等级
80000001h 扩展处理器型号, 家族,模型,平台信息
80000002h-80000004h 扩展处理器名字

写一个0选项的小程序, EAX的值是0时, 执行CPUID命令

  • EBX, string的低4 bytes
  • EDX, string 的中 4 bytes
  • ECX, string 的高 4 bytes 按照小端序(little-endian)

#cpuid.s Sample program to extract the processor Vendor ID
.section .data
    output:
	.ascii "The processor Vendor Id is 'xxxxxxxxxxxx'\n"

.section .text
.globl _start
_start:
    movl $0, %eax
	cpuid
	movl $output, %edi
	movl %ebx, 28(%edi)
	movl %edx, 32(%edi)
	movl %ecx, 36(%edi)

	movl $4, %eax
	movl $1, %ebx
	movl $output, %ecx
	movl $43, %edx
	int $0x80
	
	movl $1, %eax
    movl $0, %ebx
	int $0x80

#cpuid2.s View the CPUID Vendor ID string using C library calls
.section .data
output:
  .asciz "The processor Vendor ID is '%s'\n"
format:
.section .bss
  .lcomm buffer, 12
.section .text
.globl _start
_start:
  movl $0, %eax
  cpuid
  movq $buffer, %rsi
  movl %ebx, (%rsi)
  movl %edx, 4(%rsi)
  movl %ecx, 8(%rsi)
  movq $output, %rdi
  call printf

  xor %rdi, %rdi
  call exit

.lcomm 分配内存

静态链接(static linking)

直接将代码插入执行文件, 会导致文件很大. 并且当内存中有多个实例时浪费空间.

动态链接(dynamic linking)

动态链接可以在不将代码链接进执行文件的情况下, 当程序运行时由操作系统调用. 并且一个链接文件可以被多次复用

1
2
3

$ ld -o cpuid2 cpuid2.o -lc
$ ./cpuid2
zsh: 没有那个文件或目录: ./cpuid2

虽然链接成功了,但是运行时出了问题. 原因是链接时链接器能够解析c函数, 但并未将c函数写入最终的可执行文件(动态链接), 而是假定 运行时能够找到. 很明显现在是失败了

链接的时候必须指定,程序会在运行时加载动态库.

1
2
3

$ ld -dynamic-linker /lib/ld-linux-x86-64.so.2 -o cpuid2 cpuid2.o -lc
或者
$ ld -I /lib/ld-linux-x86-64.so.2 -o cpuid2 cpuid2.o -lc
