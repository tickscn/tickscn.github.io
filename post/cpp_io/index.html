不要拷贝和赋值 IO 对象

不能对 IO 对象拷贝和赋值. IO 对象按引用传递, 读或写一个 IO 对象会改变其状态, 所以 IO 对象不能是 const

输出缓冲区

每一个输出流都有一个缓冲区, 用来存储读写数据. 使用缓冲区可以让操作系统组合多个输出操作为一个系统级写出. 写到设置是很费时间的, 这样可以提高效率. 以下情况会立即将缓冲区写入目标

  1. 程序结束. 主函数的 return 包括清空缓冲区
  2. 缓冲区满
  3. unitbuf is set. 默认时 cerr 的 unitbuf 置 1. 所以 cerr 会立即输出
  4. 绑定了另外一个流, 会立即清空被绑定的流

文件输入输出

文件模式

  1. in, 以读的方式打开
  2. out, 以写的方式打开
  3. app, 写之前跳转到文件尾
  4. ate, 打开后立即跳转到文件尾
  5. trunc, 截断文件, 即把文件长度设为 0
  6. binary, 二进制方式

规则

  • out 只能对 ofstream 和 fstream
  • in 只能对 ifstream 和 fstream
  • trunc 只能对设置了 out 的
  • app 与 trunc 矛盾. 设置了 app 就自动设置了 out
  • out 默认是 trunc 的

#include <iostream>
#include <sstream>
using namespace std;
int main(int argc, char* argv[])
{
    string str = "111 222 333 444 555";
    istringstream record(str);
    int num;
    while (record >> num) cout << num << endl;
    return 0;
}
