引擎

  • InnoDB: 是个可靠的事务引擎. 可以用来处理大量短期事务. 其性能和崩溃后的自动恢复使得它在非事务型存储中也很流行
  • MyISAM: 表含有一个小的脚注. 表级锁限制了读/写的工作负载的性能, 所以通常用于 Web 或数据仓库中的只读或近似只读的工作中. 支持压缩表, 只读的表可以压缩来减小磁盘占用和 IO. 不支持事务
  • MEMORY: 数据保存在内存中, 速度很快. 特别适合临时表. 在功能上等同于 MyISAM

主键

  • 简单键可以是表约束或列约束. 组合键必须是表约束

简单主键

  • 列约束

    CONSTRAINT [PK_Name] PRIMARY KEY

  • 表约束

    CONSTRAINT [PK_Name] PRIMARY KEY (key_column)

组合键

CONSTRAINT [PK_Name] PRIMARY KEY (key_column) key_column 是以逗号分隔的列名

外键

  • 使用 FOREIGN KEY 或 REFERENCES 来定义外键约束.
  • 与父键可以有不同列名
  • 与父键有相同的数据类型或可隐式转为相同的数据类型
  • 实际使用中外键约束总是显式命名
  • 引用完整性检查
    1. 包含外键的表插入行, 检查外键值是否引用了父表中的一个值, 如果没有插入失败
    2. 有外键的表更新行, 检查新值
    3. 更新父表的行, 检查是否被引用. 是的话禁止更新
    4. 删除父表的行, 检查是否被引用. 是的话禁止更新

CREATE TABLE titles
  (
  title_id   CHAR(3)      NOT NULL,
  title_name VARCHAR(40)  NOT NULL,
  type       VARCHAR(10)
    CONSTRAINT type_chk
      CHECK (type IN ('biography',
        'children','computer',
        'history','psychology'))  ,
  pub_id     CHAR(3)      NOT NULL,
  pages      INTEGER
    CHECK (pages > 0)             ,
  price      DECIMAL(5,2)         ,
  sales      INTEGER              ,
  pubdate    DATE                 ,
  contract   SMALLINT     NOT NULL,
  CONSTRAINT titles_pk
    PRIMARY KEY (title_id),
  CONSTRAINT titles_pub_id_fk
    FOREIGN KEY (pub_id)
    REFERENCES publishers(pub_id),
  CONSTRAINT title_id_chk
    CHECK (
    (SUBSTRING(title_id FROM 1 FOR 1) = 'T')
    AND
    (CAST(SUBSTRING(title_id FROM 2 FOR 2)
    AS INTEGER) BETWEEN 0 AND 99)),
  CONSTRAINT price_chk
    CHECK (price >= 0.00
      AND price < 100.00),
  CONSTRAINT sales_chk
    CHECK (sales >= 0),
  CONSTRAINT pubdate_chk
    CHECK (pubdate >= DATE '1950-01-01'),
  CONSTRAINT title_name_chk
    CHECK (title_name <> ''
      AND contract >= 0),
  CONSTRAINT revenue_chk
    CHECK (price * sales >= 0.00)
  );
