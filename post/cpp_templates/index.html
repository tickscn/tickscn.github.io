定义一个模板

函数模板

函数模板是生成特定类型函数的蓝图. 由关键字 template 后跟模板参数列表(template parameter list), 参数至少有一个

1
2
3
4
5
6
7

template <typename T>
int compare(const T& lhs, const T& rhs)
{
    if (lhs < rhs) return -1;
    if (rhs < lhs) return 1;
    return 0;
}

模板参数列表类似于普通函数形参. 形参声明了变量类型, 但是没有表明如何初始化. 运行时由传递的实参初始化. 模板参数类型由运行时传递的参数确定.

实例化一个函数模板

当调用一个模板函数, 编译器使用调用时传递的参数推导模板参数. 并实例化不同的函数

1
2
3
4

cout << compare(1, 0) << endl;
vector<int> vec1;
vector<int> vec2;
cout << compare(vec1, vec2) << endl;

上面实例化了两个不同版本的 compare.

模板类型参数

模板类型可以在作用域内可以像普通类型一样使用. 每个 模板类型必须有 typename 或 class 表明. typename 与 class 在这里没有不同, 不过建议使用意义更加清晰的 typename

非类型模板参数

1
2
3
4
5

template <unsigned N, unsigned M>
int compare(const char (&p1)[N], const char (&p2)[M])
{
    return strcmp(p1, p2);
}

使用特定的类型替换 typename 或 class. 当模板实例化的时候, 非类型参数由用户提供或由编译器推导. 这些变量必须是常量表达式. 允许编器在编译时实例化模板. 非类型模板参数一般是整数, 指针, 左值引用. 其必须是常量表达式, 指针或引用的对象必须有 static 的生命周期. 当然也可以是 nulltptr.

内联和 constexpr 函数模板

和普通的函数一样, inline 和 constexpr 跟在模板参数列表后面

模板编译

编译器只在我们实例化一个模板实例时(即我们使用模板时)才会生成代码. 生成代码时编译器需要有生成代码的蓝图, 模板的实现, 所以模板代码的声明与实现必须再同一个文件中

模板类的成员函数实例化

类域中模板名简化

模板类的友元

当一个类包含一个友元声明时, 类与友元各自是否是模板是相互无关的. 如果一个类模板包含一个非模板友元, 则友元被授权可以访问所有模板实例. 如果友元自身是模板, 类可以授权给所有友元模板实例, 也可以只授权给特定实例.

// 自己的 shared_ptr
#ifndef __SHARED_POINTER_HPP_
#define __SHARED_POINTER_HPP_
#include <algorithm>
template <typename T>
class SharedPointer {
    using del_ptr = void (*)(T*);

private:
    T* p;
    del_ptr del;
    std::size_t* count;

public:
    SharedPointer(T* _p = nullptr, del_ptr _del = nullptr, std::size_t* _count = nullptr)
        : p(_p), del(_del), count(new std::size_t(_p != nullptr))
    {
    }

    SharedPointer(const SharedPointer& sp) : p(sp.p), count(sp.count), del(sp.del)
    {
        ++*count;
    }

    SharedPointer& operator=(const SharedPointer sp)
    {
        swap(sp);
        return *this;
    }

    SharedPointer(SharedPointer&&) = delete;
    SharedPointer& operator=(SharedPointer&&) = delete;

    ~SharedPointer()
    {
        if (p && --*count == 0)
        {
            del ? del(p) : delete p;
            delete count;
        }
    }

    void swap(SharedPointer& other) noexcept
    {
        using std::swap;
        swap(p, other.p);
        swap(count, other.count);
        swap(del, other.del);
    }

    void reset(T* _ptr, del_ptr _del = nullptr)
    {
        auto temp = SharedPointer(_ptr, _del);
        swap(temp);
    }

    T* get() const noexcept
    {
        return p;
    }

    std::size_t use_count() const noexcept
    {
        return *count;
    }

    T& operator*() const noexcept
    {
        return *p;
    }

    T* operator->() const noexcept
    {
        return p;
    }

    explicit operator bool()
    {
        return p != nullptr;
    }
};

#endif

template <typename F, typename T1, typename T2>
void flip1(F f, T1 v1, T2 v2)
{

void f(int v1, int& v2)
{
    cout << v1 << " " << ++v2 << endl;
}

int main()
{
    int i = 0;
    int j = 0;
    f(i, j);  // j 按引用传递, 执行完 j = 1
    flip1(f, i, j); // i 按值传, 执行为 i 不变
    /*
     * flip1(F f, T1 v1, T2 v2)
     * {// v1 = 0; v2 = 1;
     *    f(v2, v1); // v1 = 1;
     * }
     * v1 不是i的引用, 所以i的值不变
     */
}

重载和模板

模板函数可以被其它模板或非模板函数重载. 相同名字的函数必须有不同的参数数目或参数类型

  • 候选包括所有参数推导成功的模板实例化
  • 候选模板函数总是可执行的, 参数推导会排除不可执行的
  • 可执行函数按照参数转换排序
  • 如果有最好匹配, 那么匹配成功. 如果有多个函数有相等的等级
    • 如果只有一个非模板, 调用它
    • 如果全都是模板且有一个模板更专. 选择这个模板 const T& 一般来说可以匹配所有类型, 包括指针. T* 只能是指针. 当传指针时会调用后面的
    • 匹配失败
1

记得声明必要的函数. 如果没有模板, 忘记声明时会编译出错. 有模板时可以实例化相应的函数
