命令模式(Command)

0001-01-01
design-pattern

文章目录

  1. 意图: 将一个请求封装为一个对象, 从而可以用不同的请求对客户进行参数化; 对请求排队或记录请求日志
  2. 别名: 动作(Action), 事务(Transaction)
  3. 适用性:
    • 抽象出执行的操作来参数化某个对象. 是回调的面向对象替代品
    • 在不同的时刻指定, 排列和执行请求
    • 支操取消操作: Command 可存储执行前的状态, 并定义 Unexecute 来恢复状态, 可以维护一个执行列表, 通过向前向后遍历列表并分别调用 Unexecuted 和 Execute 来实现 “撤消” 和 “重作”
    • 支持修改日志: 当系统崩溃时可以重作这些操作
    • 用定义在原语操作上的高层操作构造一个系统. 如一个支持事务的信息系统. 一个事务封装了对数据的一组变动. Command 模式提供了对事务建模的一种方法
  4. 参与者
    • Command: 声明执行操作的接口
    • ConcreteCommand: 绑定一个接收者; 执行接收者相应操作
    • Client: 创建一个具体命令对象并设定它的接收者
    • Invoker: 要求 Command 执行请求
    • Reveiver: 知道如何实施一个请求相关的操作

#include <iostream>
#include <list>
using namespace std;
class Command {
    virtual void execute() = 0;
};  // 命令接口

// Receiver
class Stock {
public:
    Stock(const char* n, int q) : name(n), quantity(q) {}
    void buy()
    {
        cout << "Stock [Name:" << name << ", Quantity:" << quantity << "] bought." << endl;
    }
    void sell()
    {

        cout << "Stock [Name:" << name << ", Quantity:" << quantity << "] sold." << endl;
    }

private:
    string name;
    int quantity;
};

// ConcreteCommand

class BuyStock : public Command {
public:
    BuyStock(Stock* s)
    {
        stock = s;
    }
    void execute()
    {
        s->buy();
    }

private:
    Stock* stock;
};
class SellStock : public Command {
public:
    SellStock(Stock* s)
    {
        stock = s;
    }
    void execute()
    {
        s->sell();
    }

private:
    Stock* stock;
};

// Invoker

class Broker {
public:
    void addCommand(shared_ptr<Command> c)
    {
        comList.push_back(c);
    }
    void exeCommand()
    {
        for (auto c : comList)
        {
            c->execute();
        }
    }

private:
    list<shared_ptr<Command>> comList;
};

// Client

int main(int argc, char* argv[])
{
    Broker broker;
    broker.addCommand();
    return 0;
}
