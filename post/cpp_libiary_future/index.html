bitset

如果小于内建内建整数类型, 可以用整数类型和位运算来当位集. 标准库提供了 bitset 类可以更容易使用并且可以存储远大于整数位长的集合

定义和初始化

bitset b b 有 n 位, 每位都是 0. 构造器是编译期常量
bitset b(u) 从低位拷贝 unsigned long long 的位. 超出部分为 0
bitset b(s, pos, m, zero, one) 从 string s 的第 pos 位开始拷贝 m 位. zero 默认是'0', one 默认是 ‘1’
bitset b(cp, pos, m, zero, one)

bitset 中元素从 0 位开始.

b.set() 全设为 1

b.reset() 全设为 1
b.flip(pos ) pos 位设反
b.flip() 全反

regex 代表正则表达式类

regex_search 寻找第一个匹配的字符串
regex_replace 使用给定格式替换匹配的字符串
sregex_iterator 在 string 匹配时可以把 regex_search 结果绑定到该迭代器

regex 对象的选项

  • 一些操作

    1. regex r(reg, flag); reg 可以是 string, 迭代器对, C-style 字符串. 字符指针和长度数字.
    2. r.mark_count() r 中的子表达式
    3. r.flags() 返标志位

  • flags

    • icase: 忽略大小写
    • nosubs: 不保存子表达式匹配
    • optimize: 将时间花在构造上, 执行时更快
    • ECMAScript: 语法使用 ECMA-262
    • basic: 使用 POSIX 基础语法
    • extended: 使用 POSIX 扩展语法
    • awk: 使用 POSIX awk 语言语法
    • grep: 使用 POSIX grep 语法
    • egrep: 使用 POSIX egrep 语法

一些错误码. regex 是在运行时运算的, 可以认为 regex 不是属于 C/C++ 的一部分, 而是使用另一种语言的扩展

#include <fstream>
#include <iostream>
#include <regex>
#include <string>

using namespace std;
bool valid(const smatch);
int main()
{
    string phone = "(\\()?(\\d{3})(\\))?([-. ])?(\\d{3})([-. ]?)(\\d{4})";
    regex r(phone);
    smatch m;
    string s;
    while (getline(cin, s))
    {
        for (sregex_iterator it(s.begin(), s.end(), r), end; it != end; ++it)
        {
            if (valid(*it))
                cout << "valid: " << it->str() << endl;
            else
                cout << "not valid: " << it->str() << endl;
        }
    }
    return 0;
}

bool valid(const smatch& m)
{
    if (m[1].marched)
        return m[3].matched && (m[4].matched == 0 || m[4].str() == " ");  //括号成对出现.
    else
        return !m[3].matched && m[4].str() == m[6].str(); // 不匹配时可以返回字符串
}

随机数

C++11 之前, c++标准库和 C 语言一样使用 C 库函数 rand 来生成 0~(依赖具体系统, 不过最小为)32767 的伪有随机数. 有一些问题

  1. 许多程序需要的随机数范围与 rand 提供的不一样
  2. 一些程序需要浮点型随机数

定义在头文件 random 的随机数标准库可以解决这些问题. 该库提供了 random-number engines 和 random-number distribution classes

1

C++ 程序员应该使用 default_random_engines 来代替 rand

结果是 16807 282475249 1622650073 984943658 1144108930 470211272 101027544 1457850878 1458777923 2007237709, 而且多次运行结果相同, 好像与 C++ primer 5th 的示例结果也相同. 因为没有设置随机种子. 库定义了一些随机数引擎. 每个编译器指定其中一个为默认引擎.

  1. Engine e: 用默认种子构造
  2. Engine e(s): 使用整数类型作为种子
  3. e.seed(s): 重设种子
  4. e.min()
  5. e.max()
  6. Engine::result_type
  7. e.discard(u): 使用 unsigned long long 来跳过 u 个随机数

int main()
{
    default_random_engine e;
    normal_distribution<> n(4, 1.5);  // 期望 4, 标准差 1.5. 出现在 (-0.5, 8.5) 的概率为 99.74%
    vector<unsigned> count[9];
    for (size_t i = 0; i < 200; ++i)
    {
        unsigned r = lround(n(e));
        if (r < 9) ++count[r];
    }
    for (size_t i = 0; i != 9; ++i) cout << i << ": " << string(count[i], '*') << endl;
    return 0;
}

  • 伯努力分布 bernoulli-distribution

    \(X \sim B(1,p)\) n 为 1 的二项分布, 一次实验成功概率为 p. 失败为 1-p. 期望 p, 方差 p(1-p) C++ 新库中 bernoulli-distribution 不是模板类, 是普通类. 按照给定概率(默认为 0.5) 返回 true

多字节操作符

一次性操作一个数据块, 需要管理字符数组, 是内存泄漏的高发区.

is.get(sink, size, delim) delim 结束符, 不将其保存到 sink, 而是留在输入流中
is.getline(sink, size, delim) delim 存入 sink
is.read(sink, size) 读 size 个比特
is.gcount() 上一次无格式读取读入的比特
os.write(source, size) 将 source 中 size 比特的字符写入 os
is.ignore(size, delim) 忽略 size 个比特或到 delim. size 默认为 1, delim 默认为 eof

流的随机访问

许多流支持随机访问, 不过常用的 cin, cout, cerr, clog 不支持

seek 和 tell

操作 描述
tellg()/tellp() 返回输入/输出标志当前位置
seekg(pos)/seekp(pos) 重定位
seekg(offset, from)/seekp(offset, from) 输入流/输出流重定位到 from 偏移 offset 的位置

from 可以是

  • beg: 流的头
  • cur: 当前位置
  • end: 流的尾

流中只有一个标志位, 即使是同时支持读/写的流
