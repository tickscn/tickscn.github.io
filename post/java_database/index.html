Java 数据库连接(JDBC) API 用来使编程人员连接数据库. 并使用结构化查询语言(Structured Query Language, SQL)完成对数据的查询与更新.

JDBC 的设计

Java 为 SQL 访问提供一套 API, 同时提供一个驱动管理器, 可以通过驱动程序连接到第三方数据库

JDBC 驱动程序类型

  • JDBC 翻译为 ODBC(Open Database Connectivity): Java 8 已经不在提供 JDBC/ODBC 桥
  • Java 程序和本地代码组成, 用于与数据库的客户端进行通信
  • 纯 Java 客户端类库. 用数据库无关的协议转发数据库请求给服务器
  • 纯 Java 类库. JDBC 直接翻译为数据库相关协议

JDBC 目标

  • 通过标准的 SQL 语句和 SQL 扩展, 就可以开发 Java 数据应用
  • 数据库和数据库工具供应商可以提供底层驱动

JDBC 典型用法

  • 客户端/服务器模型中, JDBC 应该部署在客户端
  • 三层模型: 客户端-中间件-服务器. JDBC 部署在中间件

结构化查询语句(SQL)

按照惯例, SQL 关键字使用大写(只是惯例, 不是强制要求)

SELECT 语名用来查询. SELECT * FROM Books 可以查看 Books 中的所有记录. 也可以指定查询列 SELECT ISBN FROM Books. 每个 SELECT 语名都必须有 FROM 子句来指定查询的表. 可以以使用 WHERE 来指定行

1
2
3

SELECT ISBN, Price, Title
FROM Books
WHERE price <= 30

SQL 判断

  • 相等 =
  • 不等 <>

WHERE 子名也可以通过 LIKE 来实现模式匹配. % 表示 0 或多个字符, _ 表示一个字符. 字符串用单引号括起来. 字符串中间的单引号用两个单引号表示

使用 UPDATE 更改数据

1
2
3

UPDATE Books
SET Price = Price - 5.00
WHERE Title LIKE '%C++%'

管理连接, 语句, 结果集

每个 Connection 可以创建 1 个或多个 Statement. 同一个 Statement 可以执行多个不相关的命令和查询, 但是一个 Statement 最多只能有一个打开的结果集. 如果需要执行多个查询操作并同时分析结果.

至少有一种常用数据库(Microsoft SQL Server)的 JDBC 程序只允许存在一个活动的 Statement. 如果结果集相互关联, 可以使用组合查询. 使用完 ResultSet, Statement, Connection 后应立即调用 close 方法. 如果所有连接都是短时的, 可以使用带资源的 try 语句

可滚动, 可更新的结果集

一些显示用的结果集, 用户有可能希望可以在结果集上前后移动甚至移动到任意位置. 可滚动的结果集可以做到.

另外, 用户很有可能想要修改显示的内容, 可更新的结果集用来处理这种情况

可滚动的结果集

默认情况下结果集不可滚动也不可更新, 若要改变, 需要使用以下方法获取 Statement

1
2
3
4
5

Statement stat = conn.createStatement(type, concurrency);
/**
  * 预备语句
  * PreparedStatement stat = conn.prepareStatement(type, concurrency);
  */
type 解释
ResultSet.TYPE_FORWARD_ONLY 不能滚动(默认)
ResultSet.TYPE_SCROLL_INSENSITIVE 可以滚动, 对数据库更新不敏感
ResultSet.TYPE_SCROLL_SENSITIVE 可以滚动, 对数据库更新敏感
concurrency 解释
ResultSet.CONCUR_READ_ONLY 不能用于更新(默认)
ResultSet.CONCUR_UPDATABLE 可以用于更新 
1
2
3

Statement stat = conn.createStatement(ResultSet.TYPE_SCROLL_INSENSITIVE, ResultSet.CONCUR_READ_ONLY);

ResultSet st = stat.executeQuery(command);

上面生成了一个可滚动的结果集

并非所有数据库驱动都支持

  • next()
  • previous()
  • relative(n) 一次移动多行
  • absolute(n) 转到指定行号
  • getRow() 返回当前行号

可更新的结果集

1
2
3

Statement stat = conn.createStatement(ResultSet.TYPE_SCROLL_INSENSITIVE, ResultSet.CONCUR_UPDATABLE);

ResultSet st = stat.executeQuery(command);

并不是所有的查询操作都可以返回可更新结果集的, 可以调用 ResultSet 的 getConcurrency 方法来确定是否可更新

行集

构建行集

javax.sql.rowset 包提供的接口

  • CachedRowSet: 允许断开连接时操作
  • WebRowSet: 代表一个被缓存的行集, 可以保存为 XML 并移动到 Web 应用的其它层中, 只需要在该层中用另一个 WebRowSet 重新打开
  • FilteredRowSet 和 JoinRowSet: 支持轻级操作. 相当于 SELECT 和 JOIN
  • JdbcRowSet 是 ResultSet 的包装器

Java SE 7 以及后可以使用

1
2

RowSetFactory factory = RowSetProvider.newFactory();
CachedRowSet crs = factory.createCachedRowSet();

事务

将一组语句组成一个事务(transaction), 当所有语句被执行成功, 事务被提交

JDBC 事务编程

默认数据库的连接是自动提交模式, 每个 SQL 语句被执行后便立即提交, 提交后不能回滚. 要使用事务模式, 需要关闭该默认值

1

conn.setAutoCommit(false);

然后向之前一样操作, 如果执行了所有语句且没有出错, 就可以提交了

1
2

conn.commit();
// conn.rollback(); 出错了调用
