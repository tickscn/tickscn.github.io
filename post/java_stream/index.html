通过流可以说明想要完成什么任务, 而不是如何实现它

迭代与流

处理集合时常常需要遍历它的元素, 并在每个元素上执行某个操用.

String contents = new String(Files.readAllBytes(Paths.get("text.txt")), StandardCharsets.UTF_8);
List<String> words = Arrays.asList(contents.split("\\PL+"));

// 迭代
long count = 0;
for(String w:words){
    if(w.length > 10) count++;
}

// 流
long count = words.stream().filter(w->w.lenght>10).count();
  1. 流并不存储元素, 这些元素有可能存于底层的数组, 也有可能按需生成
  2. 流的操作不改变数据源
  3. 流的操作惰性执行, 只有需要时才执行

操作流的典型流程

  1. 创建一个流
  2. 指定将初始流转换为其它流的操作, 可能需要好多步
  3. 终止操作, 产生结果

流的创建

集合 Collection 接口的 stream 方法可以将任合一个集合转换为一个流. 普通数组可以用静态的 Stream.of 方法

1

Stream<String> words = Stream.of(contents.split("\\PL+");

使用 Array.stream(array, from, to) 可以用子数组创建流

generate 方法可以按某个方法生成无限流

1

Stream<Double> randoms = Stream.generate(Math::random);

iterate 方法可以生成无限序列

1

Stream<BigInteger> integers = Stream.iterate(BigInteger.ZERO, n->n.add(BigInteger.ONE);

filter, map, flatMap

  • filter: 与某种要求匹配而派生新的流
  • map: 按某种方法转换流中的元素
1

Stream<String> lowerwords = words.stream().map(String::toLowerCase);
  • flatMap: map 有时会产生流的流, 如果想要摊平成一维的流, 可以使用 flatMap

抽取子流和连接流

  • stream.limit(n) 会返回一个新的截断在 n 的流, 如果原流长度小于 n, 等于原来的流
  • stream.skip(n) 跳过前 n 个元素
  • Stream 类的静态方法 contact 可以连接两个流

其它流转换

  • distinct 返回剔除重复元素的流
  • sorted 排序的流. 流中元素应是 Comparable 的, 或者可以传入一个 Comparator
  • peek 会产生一个元素与原来相同的新流, 但是每获取一个元素, 都会调用一个函数

简单约简

约简是一种 终结 操作, 可以将流约简为可以使用的非流值

  • count 方法返回流中元素数量
  • max, min 方法返回流中最大或最小值. 具体的说返回的是 Optional类型
  • findFirst 返回集合的第一个值
  • findAny 可以返回任何一个匹配, 可以并行处理
  • allMatch 全部匹配
  • noneMatch 没有匹配
  • anyMatch 存在匹配

Optional 类型

Optional 要么包装了 T 的对象, 要么没有包装任何对象

如何使用

不存在值时可以有替代物, 只有存在值时才使用值

  • 使用默认值
    • 字符串 optionalString.orElse("")
    • 通过调用代码获取 optionalString.orElseGet( xxx ); 只有需要时才执行
    • 抛出异常 optionalString.orElseThrow(xxx::new)
  • 只有存在值时才使用值
    • ifPresent 会接收一个函数, 当该可选值存在其会被传递给函数, 否则什么都不会发生
    • map 会把结果存入返回值

不适合使用 Optional 值的地方

  • get() 存在时返回值
  • isPresent() 存在值时返回 true

创建 Optional 值

  • of
  • ofNullable
  • empty

用 flatMap 来构建

如果方法 f 可以产生 Optional 而 T 有方法 g. 可以这样连续调用

1

s.f().flatMap(T::g)

收集结果

  • iterator 方法产生旧式的迭代器
  • forEach 方法, 在并行流上会以任意顺序遍历元素
  • forEachOrdered 方法, 按顺序遍历
  • toArray 获得由流中元素构成的数组, 返回的是 Object[], 如果想要具体的数组, 可以传递类型构造器
  • collect 接收 Collector 接口的对象, Collectors 中有许多静态工厂方法
    • stream.collect(Collectors.totLis());
    • stream.collect(Collectors.totSet());
    • stream.collect(Collectors.toCollection(xxx:new));
    • stream.collect(Collectors.joining());
    • stream.collect(Collectors.joining(",")); 加分隔符
    • stream.collect(Collectors.xxxSummaryStatistics()); 加分隔符

收集到映射表中

toMap 避免冲突, 传递第三个参数

群组和分区

将具有相同特性值群聚成组可以用 GroupingBy

1
2

Stream<Locale> locales = Stream.of(Locale.getAvailableLocales());
Map<String, List<Locale>> countryToLocales = locales.collect(Collectors.groupingBy(Locale::getCountry));

分类函数是断言函数(返回 boolean)时, partitioningBy 比 groupingBy 更高效

下游收集器

groupingBy 会产生一个映射表, values 是 List, 若想以某种方法处理该列表, 可以用下游收集器

1

Map<String, Set<Locale>> localesSet = locales.collect(Collectors.groupingBy(Locale::getCountry, Collectors.toSet()));

常用收集器

  • counting 产生收集到的元素个数
  • summingInt/Double/Long 接收一个引元, 将引元应用到下游元素并产生它们的和
  • maxBy/minBy. 接收一个比较器, 产生最大/最小值

Long product = values.stream().reduce(1, (x, y)->x * y);

基本类型流

IntStream, LongStream, DoubleStream 直接存储基本值 int, long, double 而无需使用包装器同样可以使用 generate, iterate 方法, IntStream 和 LongStream 有 range 和 rangeClosed 方法可以产生步长为 1 的整数范围

并行流

  • parallelStream()
  • stream.parallel()
