基础概念

重载运处符是特殊的函数: 定义 operator symbol(). 一元运算符有一个参数, 二元运算符有两. 二元运算符的左操作数是第一个参数, 右操作数是第二个参数. 如果运算符是成员函数, 第一个 this. 运算符函数必须是成员函数或致少有一个参数是 class type. 可以重载的运算符

+ - * / % ^
& | ~ ! , =
< > <= >= == !=
« » ++ – && ||
+= -= *= /= %= ^=
&= |= <<= >>= [] ()
-> ->* new new [] delete delete []

不可以重载的运算符
:: .* . ?:

直接调用载运算符

  1. 像运算符一样用 data1 + data2
  2. 像函数一样用 operator+(data1, data2)

不应重载运算符

一些运算符保证了操作数的运算顺序. 而重载运算符的本质是函数, 无法保证参数的运算顺序. 特别的, 逻辑运算符 AND &&, OR ||, 逗号运算符 , 不再保证顺序. 所以 && 和 || 不再有短路特性. 因此, 最好不重载这些运算符

定义应与内建运算符的意义一致

  • 如果该类有输入输出, 左移或右移动算符的行为应与内建类型一致
  • 如果该类需要判断相等, 重载 operator==. 一般来说, 还应该 重载 operator!=
  • 如果类有排序的需要, 重载 operator<. 同时应该重载所有关系运算符

选择成员函数或是非成员函数

  • 赋值(=), 下标([]), 调用(()), 成员访问箭头(->) 必须 是成员函数
  • 复合赋值运算 应该 是成员函数, 虽然没有强制要求
  • 改变对象状态或与给定类型紧密绑定的 应该 是成员函数. 如递增, 递减, 解引用
  • 对称操作符, 应该 是非成员函数

输入输出操作符

重载输出运算符 <<

第一个参数是输出流的非常量引用, 第二个参数是类的常引用, 返回它的第一个参数的引用

1

ostream& operator<<(ostream& os, const T& cls);

输出改量少的格式化

为了行为一致, 输出做最少的格式化, 一般不输出新行

IO 操作必须是非成员函数

应为通常来说类处于 IO 操作符的右边, 应定义为友元非成员函数

可调用对象有自己的类型. 但是不同类型的可调用对象有可能有相同的调用签名–返回值和必须传递的参数类型. 调用签名和函数类型相一致

都有相同的调用签名 int(int,int). 我们可能希望使用这些对象来构建软件. 可以建立一个函数表(function table)来存储这些对象的 “指针”. 当程序需要执行一个特定的对象时, 可以在表中查找. c++ 使用 map 可以很容易的实现这个函数表. 如果只是函数对象, 可以这样声明

1

map<string, int (*)(int, int)> binops;

然后可以添加

1

binops.insert("+", add);

不过这样的 binops 无法添加 mod 和 div. 因为这两者有自己的类型, 与 binops 无法匹配

请避免过度使用

类型转换有可能产生另人惊讶的结果

实际上, 类很少提供转换运算符. 如果自动发生了转换, 用户有可能会产生困扰. 不过, 有一个例外: 转换为 bool 并不少见. 在之前的标准中, 转换为 bool 有一个问题: 因为 bool 是数学类型. 被转为 bool 的类对象可以用在任何期待数学类型的地方.

1
2

cin << i;  // 我们希望这里是错误的, 但是在老的标准下, cin 转为 bool使其合法

避免多义转换

如果一个类有多个转换操作, 请确保只有一种方法将类对象转为目标对象. 不然代码会有岐义. 这是有两种产生多转换的情况

  1. 两种类可以相互转换
  2. 转换源或目标有相关转换, 如转为多种数学类型
